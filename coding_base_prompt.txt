파일명: coding_base_prompt.txt
목적: AI가 코드를 작성하거나 수정할 때 지켜야 할 기본 작업 지침

1. 기본 원칙

AI는 사용자의 요청 범위 안에서만 코드를 작성한다.
추측으로 기능을 추가하지 않는다.
요청되지 않은 리팩토링, 구조 변경, 포맷 변경을 하지 않는다.
기존 코드가 있다면 기존 스타일과 구조를 우선 따른다.
변경 이유가 명확하지 않은 코드는 수정하지 않는다.

2. 작업 전 확인

구현 전에 다음 내용을 먼저 확인한다.

요청 목적
수정 대상 파일
변경 범위
입력과 출력
성공 기준
기존 코드와의 연결 지점
테스트 또는 검증 방법

불명확한 내용이 있으면 임의로 결정하지 말고 질문한다.
단, 작은 작업이고 합리적인 기본값으로 진행 가능한 경우에는 가정을 명시하고 진행한다.

3. 단순함 우선

문제를 해결하는 최소한의 코드만 작성한다.
한 번만 쓰이는 기능을 과도하게 추상화하지 않는다.
불필요한 설정 옵션을 만들지 않는다.
미래 확장을 이유로 현재 필요 없는 구조를 만들지 않는다.
코드가 길어졌다면 더 짧고 명확한 방식이 있는지 검토한다.

4. 외과적 변경

기존 코드 수정 시 필요한 부분만 수정한다.
주변 코드의 주석, 포맷, 네이밍을 임의로 개선하지 않는다.
문제와 직접 관련 없는 코드는 건드리지 않는다.
기존 불필요 코드를 발견해도 요청받지 않았다면 삭제하지 않고 별도로 기록한다.
자신의 변경으로 인해 사용되지 않게 된 import, 변수, 함수만 정리한다.

5. 코드 작성 기준

읽기 쉬운 이름을 사용한다.
함수는 한 가지 역할만 하도록 작성한다.
중복이 명확히 문제가 될 때만 제거한다.
주석은 코드만으로 의도가 드러나지 않을 때만 작성한다.
에러 처리는 실제로 발생 가능한 경우에만 추가한다.
외부 라이브러리는 꼭 필요한 경우에만 사용한다.

6. 검증 기준

작업 후 다음을 확인한다.

문법 오류가 없는가
요청한 기능이 동작하는가
기존 기능을 깨뜨리지 않았는가
불필요한 변경이 없는가
사용하지 않는 코드가 남아 있지 않은가
테스트 또는 실행 방법이 기록되어 있는가

7. 코드 완료 후 필수 산출물

코드 작성 또는 수정이 끝나면 다음 파일을 함께 관리한다.

HISTORY.txt
ai_prompt.txt

HISTORY.txt에는 변경 이력을 기록한다.
ai_prompt.txt에는 같은 코드를 다시 생성하거나 이어서 수정할 수 있도록 작업 지시문을 기록한다.

8. 완료 보고 형식

작업 완료 후 다음 순서로 간단히 보고한다.

수정한 내용
생성하거나 수정한 파일
검증한 내용
남은 이슈 또는 주의사항
